# ХЕХЕ 😂

Магазин мемных футболок. React + Vite + Framer Motion.

```bash
npm install
npm run dev      # http://localhost:5173
npm run build
```

## Подключение Supabase (для авторизации)

Без ключей сайт открывается и работает как витрина — просто вход, кошелёк,
заказы и избранное выключены, вверху висит предупреждение.

1. Заведи бесплатный проект на [supabase.com](https://supabase.com).
2. **SQL Editor** → вставь и выполни весь `supabase/schema.sql`.
   Скрипт создаёт таблицы `profiles`, `orders`, `favorites`, включает RLS
   и добавляет функции `topup()` и `checkout()`.
3. **Project Settings → API** → скопируй Project URL и ключ `anon public`.
4. Скопируй `.env.example` в `.env` и подставь их:
   ```
   VITE_SUPABASE_URL=https://xxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJhbGciOi...
   ```
5. Перезапусти `npm run dev` (Vite читает `.env` только при старте).

**Вход через Google** (опционально): Authentication → Providers → Google,
включить и вписать Client ID/Secret из Google Cloud Console.
Без этого работает вход по почте и паролю.

**Подтверждение почты**: по умолчанию Supabase шлёт письмо со ссылкой.
Для быстрых тестов можно выключить в Authentication → Sign In / Providers →
Confirm email, тогда аккаунт активен сразу.

### Почему деньги считаются в базе, а не в браузере

Списание баланса и создание заказа делает SQL-функция `checkout()` с
`security definer`: она сама пересчитывает сумму, берёт `for update` на строку
профиля (два одновременных заказа не спишут один баланс дважды) и падает с
`INSUFFICIENT_FUNDS`, если денег не хватает. Прямой INSERT в `orders`
политиками запрещён, так что заказ нельзя дописать в обход проверки.

## Что внутри

- **Авторизация** — почта+пароль и Google через Supabase. Профиль заводится триггером при регистрации, стартовый баланс 25 000 ₸.
- **Кабинет** — профиль (имя, аватар-эмодзи), история заказов, избранное.
- **Двуязычие RU / ҚАЗ** — переключается всё, включая надписи на самих футболках. Выбор языка сохраняется.
- **ХЕХЕ Wallet** — баланс в тенге, пополнение, оплата, кэшбэк 7% ХЕХЕ-коинами. Живёт в базе, считается на сервере.
- **ЛОЛ-метр** — рейтинг смешноты вместо звёздочек, он же принцип сортировки.
- **Редкость принтов** — Обычный / Редкий / Легендарный, работает как фильтр.
- **Два вида карточек** — рисованные в SVG (принт подстраивается под длину текста и язык) и фотографии из `public/`.
- **Таймер дропа**, бегущая строка, «Мне повезёт 🎲», эмодзи-хвост за курсором.

## Как добавить футболку

Один объект в `PRODUCTS` (`src/data.js`): `price`, `rarity`, `lol`, цвета `color`/`ink`
и блоки `ru` / `kk` с `title`, `print`, `sub`.

Дальше два варианта:

- **Рисованная** — задай `art` (ключ мини-иллюстрации из `Tshirt.jsx`) и `print`.
  `print` разбивается по `\n`, кегль подбирается автоматически.
- **Фотография** — положи файл в `public/` и укажи `photo: '/имя-файла.jpg'`.
  Тогда `art` и `print` не нужны, показывается сама картинка.
